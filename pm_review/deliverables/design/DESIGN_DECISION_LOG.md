# 디자인 결정 로그 (Design Decision Log)

본 문서는 **Phase 8-0 (Product Design Sprint & UX Architecture Foundation)** 과정에서 결정된 주요 시각 스타일, UI/UX 패턴, 컴포넌트 아키텍처적 의사결정 사항(ADR)을 보존한다.

---

## 1. [DDL-01] 슬레이트(Slate) 기반의 프리미엄 다크 테마 기본 채택
* **결정 배경**: AI 분석 비서 플랫폼으로서의 정체성을 부각하고, 장시간 데이터를 스캔하는 판매자들의 눈의 피로를 최소화하기 위함.
* **의사결정**:
  - 기본 배경은 `hsl(222.2, 84%, 4.9%)` (어두운 감청색 가미 슬레이트)를 채택하며, 카드는 `hsl(222.2, 84%, 7.5%)`로 구분하여 단차를 부여함.
  - 액센트 컬러로는 미래지향적인 Royal Indigo(`hsl(263.4, 70%, 50.4%)`)를 적극 차용.
* **영향도**: 모든 shadcn/ui 컴포넌트의 기본 CSS 및 `design-tokens.json`에 반영됨.

---

## 2. [DDL-02] 상품 상세 뷰 내 로컬 탭(Local Tab) 네비게이션 적용
* **결정 배경**: 개별 상품에 매핑되는 하위 정보(결핍 분석 ➡️ 기획 스토리보드 ➡️ 성과 환류) 조회 시, 매번 전체 화면 리로드 방식의 페이지 라우팅을 수행하면 데이터 로딩 시간 지연과 흐름 단절을 초래함.
* **의사결정**:
  - `/products/[id]` 진입 시 단일 최상위 컨테이너 레이아웃을 공유하고, 하단 컨텐츠 영역은 Next.js의 Dynamic Routing을 사용하되 UI상으로는 **로컬 탭(Tabs) 컴포넌트**로 구성하여 즉각적 전환 효과(Tab Transitions)와 통일감을 제공함.
* **영향도**: `INFORMATION_ARCHITECTURE.md` 및 `SCREEN_INVENTORY.md` 설계 표준으로 수립.

---

## 3. [DDL-03] AI 스토리보드 단계적 Progressive Disclosure 적용
* **결정 배경**: 8단계 상세페이지 기획안을 단일 종 스크롤 레이아웃에 전부 렌더링할 경우, 텍스트와 프롬프트 데이터 과다로 인하여 사용자의 스크롤 피로와 심각한 인지 과부하(Cognitive Load)가 확인됨.
* **의사결정**:
  - 8개 단계를 타임라인 피드 형태로 배열하고, **아코디언(Accordion) 컴포넌트**를 결합함. 기본 상태에서는 각 단계의 요약 권장 한 줄 문구만 보여주고, 사용자가 추가 상세 행동 지침을 원할 때에만 클릭하여 텍스트를 전개하도록 가이드함.
* **영향도**: `UX_GUIDELINE.md` 및 `UX_PATTERN.md`에 반영됨.

---

## 4. [DDL-04] 감사 이력 조회 시 페이지네이션(Pagination) 단일 표준 채택
* **결정 배경**: 감사 데이터는 특정 보안 감사 사건(Event)을 추적하고 비교하는 정량 분석이 주 목적이므로, 페이스북 스타일의 무한 스크롤(Infinite Scroll) 적용 시 사용자 스크롤 위치 보존이 불가능하고 데이터 역추적이 어려움.
* **의사결정**:
  - 감사 로그 테이블 하단에는 항상 20개 행씩 잘라서 전달하는 페이지네이션 네비게이터를 표준 구성함.
* **영향도**: `UX_PATTERN.md` 및 `SCREEN_INVENTORY.md` 반영.
