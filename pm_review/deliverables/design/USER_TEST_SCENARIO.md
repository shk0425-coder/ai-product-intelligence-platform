# USER_TEST_SCENARIO.md

# 사용자 테스트 시나리오 (User Usability Test Scenarios)

본 문서는 프론트엔드 UI 프로토타입 완료 후, 실제 스마트스토어 판매자가 시스템을 사용하며 사용 편의성을 평가하기 위한 **표준 사용자 테스트 태스크 시나리오**이다.

---

## 1. 테스트 준비 및 목적
* **테스트 대상**: 네이버 스마트스토어 신입 셀러 및 1인 기업 셀러 3명.
* **평가 목적**:
  - 데이터 분석 지식 없이도 첫 분석 및 기획 스토리보드를 성공적으로 도출하는가?
  - 비동기 로딩 대기 시간 동안 이탈이나 불안감을 느끼지 않는가?
  - 에러 상황 및 한도 임계치 알림의 대응 방법을 직관적으로 이해하는가?

---

## 2. 테스트 시나리오 시트 (User Tasks)

### 2.1. 태스크 1: 신규 상품 검색 및 시장성 분석 실행
* **목표**: 판매 희망 키워드 `친환경 텀블러`를 대상으로 네이버 쇼핑 리뷰 분석을 수행한다.
* **시나리오 스크립트**:
  1. 홈 대시보드 화면에서 [시장 분석 시작] 카드를 찾아 클릭하세요.
  2. 키워드 입력 필드에 "친환경 텀블러"를 입력하고, 분석 리뷰 건수를 `100건`으로 조절하세요.
  3. [AI 분석 시작] 버튼을 누른 후, 화면에 출력되는 실시간 수집 및 분석 진행 프로세스를 확인하며 분석 완료 때까지 대기하세요.
* **평가 기준**:
  - 검색 페이지 진입 경로를 5초 이내에 찾는가?
  - 분석 중일 때 화면 로딩 바와 현재 단계를 직관적으로 이해하고 브라우저를 닫지 않고 기다리는가?

### 2.2. 태스크 2: JTBD 결핍 확인 및 상품 기획안 승인
* **목표**: 분석 완료 후 도출된 고객 JTBD(결핍)를 검토하고, AI 8단계 스토리보드를 승인한다.
* **시나리오 스크립트**:
  1. 분석 리포트 화면에서 소비자가 가장 불만을 느끼는 1순위 결핍 요인과 대표 구매평 인용구를 확인하세요.
  2. [AI 기획안 생성] 탭으로 전환하여 8단계 상세페이지 스토리보드 흐름을 검토하세요.
  3. 기획안의 썸네일 이미지 시안을 크게 확대하여 확인한 후, [기획안 최종 승인] 버튼을 클릭하세요.
* **평가 기준**:
  - 소비자의 핵심 불편 사항(결핍) 요약 카드와 상세 대표 리뷰 글을 올바르게 매치하여 인지하는가?
  - 8단계 기획안을 차례대로 스크롤하며 아코디언 토글을 열어 상세 내용을 확인하는가?
  - 승인 버튼을 누른 뒤 시안 이미지가 매핑되는 성공 결과를 정확히 인지하는가?

### 2.3. 태스크 3: 감사 로그 필터 및 CSV 내보내기
* **목표**: 어제 발생한 경고(Warning) 등급의 운영 감사 내역을 찾아서 CSV 파일로 내보낸다.
* **시나리오 스크립트**:
  1. 좌측 메뉴에서 [운영 센터] ➡️ [보안 감사] 탭으로 이동하세요.
  2. 검색 필터에서 심각도를 `Warning`으로 지정하고 검색 버튼을 누르세요.
  3. 필터링된 감사 내역을 확인하고 [CSV 내보내기] 버튼을 눌러 로컬 다운로드가 완료되는지 확인하세요.
* **평가 기준**:
  - 운영 센터 메뉴에 정상적으로 접근하여 필터를 조작하는가?
  - CSV 다운로드 결과를 인지하고 파일을 여는가?
