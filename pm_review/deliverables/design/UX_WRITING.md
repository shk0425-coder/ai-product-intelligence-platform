# UX 라이팅 및 용어 정의서 (UX Writing)

본 문서는 **AI Product Intelligence Platform**의 화면 전반에서 사용되는 텍스트의 톤앤매너(Tone and Manner), 용어 변환 규칙 및 에러/경고 문구 양식을 정의한다.

---

## 1. 보이스 앤 톤 (Voice & Tone)

우리의 보이스 앤 톤은 **"전문적이되 권위적이지 않고, 신뢰할 수 있으며 친절한 파트너"**의 모습을 지향한다.

| 원칙 | 설명 | 나쁜 예시 (Bad) | 좋은 예시 (Good) |
|:---|:---|:---|:---|
| **직관성 (Directness)** | 불필요한 테크니컬 은어나 수식어를 배제하고, 사용자가 즉시 이해할 수 있는 비즈니스 용어로 전달한다. | "Zod 스키마 파싱 오류로 인해 백엔드 API Validation에 실패하였습니다." | "입력하신 상품 정보 형식이 올바르지 않습니다. 다시 확인 후 시도해 주세요." |
| **친절함 (Supportiveness)** | 단순 경고나 오류 상황에서도 해결방안을 부드럽게 권장한다. | "작업 실패. Quota 임계치 초과함." | "이번 달 AI 기획 호출 한도(Quota)를 초과하였습니다. 플랜을 업그레이드하거나 설정에서 한도를 늘릴 수 있습니다." |
| **보이지 않는 AI (Jargon-free)** | 머신러닝 및 AI 내부 파이프라인의 명칭을 사용자에게 노출하지 않는다. | "Gemini Flash 모델이 Temperature 0.2로 JTBD Embedding을 완료함." | "AI 분석 비서가 230개의 소비자 평점을 분석해 핵심 불편 사항을 도출했습니다." |

## 2. 핵심 도메인 용어 변환 사전 (Terminology Dictionary)

시스템 내부 데이터베이스/코드 구조의 영문 키워드들을 UI 화면상에서 어떻게 국문 친화적으로 번역하여 표기할지 정의한다.

| 개발자 용어 (DB / API) | 사용자 UI 노출 용어 | 번역 근거 |
|:---|:---|:---|
| `workspace_id` | **워크스페이스** (또는 스토어/조직) | 테넌트 격리 개념을 직관적으로 인지. |
| `jtbd_insights` | **소비자 핵심 결핍 (JTBD)** | Jobs-to-be-done의 이론적 설명 대신 직관적인 '결핍' 강조. |
| `product_strategies` | **AI 상품 기획안** | 기획의 최종 산출물 성격을 부각. |
| `creative_briefs` | **상세페이지 컨셉 시안** | 디자인 썸네일과 문구 시안을 아우르는 명칭. |
| `audit_logs` | **운영 로그 (또는 감사 이력)** | IT 용어 '감사(Audit)'를 직관적인 '운영 기록'으로 변환. |
| `usage_metering` | **실시간 사용량** | Metering의 계량 단위를 알기 쉽게 번역. |
| `resilience` | **시스템 복구 자동화** | 탄력성이라는 추상적 용어 대신 '자동 복구'로 인지. |

## 3. 상황별 표준 문구 템플릿 (Standard Copy Templates)

### 3.1. 로딩 상태 문구 (Loading / Progress States)
* **네이버 리뷰 수집 시**: `스토어에서 구매평을 수집하고 있습니다. 잠시만 기다려 주세요 (약 20초 소요)`
* **AI 기획안 생성 시**: `AI 비서가 기획 아이디어를 조합하고 있습니다. 문장 구조를 가다듬는 중입니다.`

### 3.2. 에러 알림 문구 (Error Notification States)
* **네트워크 순단**: `네트워크 연결이 일시적으로 끊어졌습니다. 연결 상태를 확인하고 [다시 시도]를 눌러주세요.`
* **권한 없음 (격리 위반)**: `해당 워크스페이스의 정보에 접근할 권한이 없습니다. 관리자에게 워크스페이스 멤버 권한을 요청해 주세요.`
