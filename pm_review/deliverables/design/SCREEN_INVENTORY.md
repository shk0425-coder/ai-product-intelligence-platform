# SCREEN_INVENTORY.md

# 화면 목록 정의서 (Screen Inventory)

본 문서는 **AI Product Intelligence Platform**에 구현될 모든 화면의 세부 명세와 구성 요소, 그리고 인터랙티브 상태를 규정한다.

---

## 1. 공통 레이아웃 (Common Global Layout)
* **GNB (Global Navigation Bar)**: 로고, 메인 메뉴 링크 5개, 워크스페이스 스위처, 알림 아이콘, 프로필 드롭다운.
* **LNB (Local Navigation Bar - 상품 상세 전용)**: 개별 상품 선택 시 노출되는 좌측 서브 메뉴 (개요, JTBD 분석, 기획 스토리보드, 성과 환류).
* **Footer**: 카피라이트, 시스템 상태 요약 배지 (정상 작동 / 유지보수 모드 등).

---

## 2. 화면별 세부 인벤토리 (Screen Directory)

### 2.1. 홈 포털 대시보드 (Home Dashboard)
* **주요 요소**:
  - 워크스페이스 정보 요약 카드 (이름, 테넌트 ID).
  - Quota 한도 대비 실시간 사용량 프로그레스 바.
  - 최신 감사 로그 목록 및 심각도 배지 (Info, Warning, Critical).
  - 퀵 온보딩 카드 ("첫 상품 시장 분석하기").
* **인터랙티브 상태**:
  - **Loading**: 스켈레톤 카드로 메트릭 영역 표시.
  - **Empty**: 등록된 상품이 없을 시 온보딩 카드만 크게 표시.
  - **Error**: API 호출 실패 시 "메트릭을 불러올 수 없습니다" 안내 및 새로고침 버튼.
* **사용자 액션**:
  - 워크스페이스 스위처 클릭 ➡️ 워크스페이스 변경.
  - "분석 시작" 버튼 클릭 ➡️ 신규 상품 등록 화면 이동.

### 2.2. 신규 상품 소싱 검색 & 분석 실행 (Product Search & Scraper)
* **주요 요소**:
  - 네이버 쇼핑 카테고리/키워드 검색 바.
  - 타겟 수집 리뷰 수 설정 슬라이더 (최소 50개 ~ 최대 500개).
  - "AI 분석 시작" 버튼.
  - 실시간 분석 단계별 프로그레스 다이어그램.
* **인터랙티브 상태**:
  - **Loading (진행 중)**: 크롤러 구동 및 분석 상태를 3단계(`수집 중` ➡️ `결핍 추출 중` ➡️ `완료`)로 실시간 프로그레스 바 표시.
  - **Success**: 분석 완료 시 "성공적으로 분석되었습니다" 토스트 및 상품 상세로 즉시 리다이렉트.
  - **Error**: 크롤러 차단 또는 네트워크 실패 시 에러 사유 노출 및 재시도 버튼.
* **사용자 액션**:
  - 키워드 입력 및 슬라이더 조절.
  - 분석 실행 및 취소.

### 2.3. 상품 목록 및 상태 그리드 (Product List)
* **주요 요소**:
  - 상품명, 타겟 카테고리, S~D 기획 등급, 최종 기획일자, 성과 점수 테이블 그리드.
  - 검색 및 카테고리 필터 드롭다운.
  - "신규 등록" 플로팅 액션 버튼.
* **인터랙티브 상태**:
  - **Loading**: 테이블 행 스켈레톤 로더.
  - **Empty**: "등록된 상품이 없습니다. 첫 상품을 등록하세요." 안내 및 퀵 링크.
* **사용자 액션**:
  - 상품 행 클릭 ➡️ 상품 상세 화면으로 이동.
  - 필터 선택 ➡️ 테이블 즉시 필터링.

### 2.4. JTBD 분석 & 결핍 인사이트 (JTBD Analysis View)
* **주요 요소**:
  - 수집된 리뷰 데이터 요약 카드 (총 평점, 긍/부정 반응 비율).
  - AI가 분류한 핵심 JTBD (고객 결핍) 카드 그리드.
  - 결핍별 중요도(가중치) 및 고객 대표 인용구 토글.
* **인터랙티브 상태**:
  - **Loading**: 감성 분석 및 결핍 추출 스피너.
  - **Success**: S~D 규칙 엔진 등급 스코어 카드 렌더링.
* **사용자 액션**:
  - JTBD 카드 클릭 ➡️ 대표 리뷰 원문 상세 모달 오픈.

### 2.5. AI 8단계 스토리보드 & 이미지 시안 (Product Strategy & Creative Brief)
* **주요 요소**:
  - 8단계 상세페이지 스토리보드 타임라인.
  - 각 단계별 AI 추천 문구 및 이미지 컨셉 프롬프트.
  - FLUX API 연동 이미지 시안 썸네일 그리드.
  - "기획안 승인 및 출시 완료" Primary Action Button.
* **인터랙티브 상태**:
  - **Loading**: 스토리보드 JSON 구조 및 이미지 렌더링 스피너.
  - **Waiting (승인 대기)**: 대표/매니저 승인 버튼 노출.
* **사용자 액션**:
  - 이미지 썸네일 클릭 ➡️ 고해상도 시안 이미지 라이트박스 팝업.
  - "승인" 또는 "반려" 피드백 전송.

### 2.6. 성과 지표 & Closed-Loop 학습 환류 (Performance & Feedback View)
* **주요 요소**:
  - 누적 매출액, 고객 평점, 긍정 반응률 시각화 배지.
  - AI가 학습한 성공 패턴 및 보완해야 할 프롬프트 피드백 카드.
  - "실적 데이터 갱신" 동기화 버튼.
* **인터랙티브 상태**:
  - **Loading**: 피드백 연산 가중치 재계산 로딩.
* **사용자 액션**:
  - 수동 성과 데이터 입력 및 갱신 API 호출.

### 2.7. 중앙 감사 로그 (Audit Center View)
* **주요 요소**:
  - 감사 로그 필터 패널 (기간, Severity, Workspace, Action).
  - 테이블 뷰 (타임스탬프, 행위자, 액션, 변경점 요약).
  - "감사 데이터 CSV/JSON 다운로드" 버튼.
* **사용자 액션**:
  - 행 클릭 ➡️ JSON 상세 변경 스냅샷(이전값 vs 현재값) 비교 팝업 모달.
  - 다운로드 버튼 클릭 ➡️ 브라우저 파일 즉시 다운로드.

### 2.8. 복구 및 탄력성 모니터 (Resilience Hub)
* **주요 요소**:
  - Redis Circuit Breaker 실시간 상태 배지 (`CLOSED`, `OPEN`, `HALF_OPEN`).
  - 현재 실행 중인 분산 락 및 실패 작업 큐 (Dead Letter Queue) 리스트.
  - "유지보수 Read Only 모드" 스위치 토글.
* **사용자 액션**:
  - DLQ 작업 "수동 재시도" 클릭 ➡️ 복구 API 기동.
  - Read Only 모드 켜기/끄기 (보안 패스워드 재검증 필요).
