# COMPONENT_CATALOG.md

# 공통 컴포넌트 카탈로그 (Component Catalog)

본 문서는 **AI Product Intelligence Platform**의 화면들을 조립하는 데 사용될 핵심 공통 컴포넌트 목록과 각 컴포넌트의 기능 명세를 규정한다.

---

## 1. 전역 레이아웃 및 내비게이션 (Layout & Navigation)

### 1.1. `Sidebar`
* **설명**: 메인 메뉴 탐색을 제공하는 좌측 고정 사이드바.
* **구성 요소**: 로고, 워크스페이스 스위처, 메인 링크 목록(홈, 상품, 감사, 탄력성, 설정), 프로필 드롭다운.
* **상태**: 데스크톱 확장 상태, 모바일 축소(아이콘만 표시) 상태.

### 1.2. `Header`
* **설명**: 상단 내비게이션 툴바.
* **구성 요소**: 현재 브레드크럼(Breadcrumb), 실시간 시스템 상태 배지, 알림 벨 아이콘.

---

## 2. 모니터링 및 메트릭 컴포넌트 (Telemetry & Metrics)

### 2.1. `QuotaProgress`
* **설명**: 워크스페이스의 AI 호출 횟수 또는 한도 사용률을 시각화하는 위젯.
* **구성 요소**: 한도명, 실시간 사용 비율 수치 (`85 / 100회`), 커스텀 색상 상태 바.
* **상태 변화**:
  - `Usage < 70%`: Emerald 녹색 바.
  - `70% <= Usage < 90%`: Amber 주황색 바 (경고).
  - `Usage >= 90%`: Rose 적색 바 (임계값 임박).

### 2.2. `GradeBadge`
* **설명**: 상품 기획 가치의 최종 판정 등급(S~D)을 출력하는 배지 컴포넌트.
* **상태**:
  - `S` / `A` 등급: Emerald 배경색 + 녹색 텍스트.
  - `B` / `C` 등급: Indigo 배경색 + 보라색 텍스트.
  - `D` 등급: Muted 회색 배경색 + 회색 텍스트 (기획 보류 권장).

---

## 3. 도메인 특화 컴포넌트 (Domain Specific Components)

### 3.1. `JtbdInsightCard`
* **설명**: AI가 요약한 단일 소비자 JTBD(결핍 요인) 요약 카드.
* **구성 요소**: 결핍 카테고리, 결핍 요약문, 가중치 배지, 대표 소비자 인용구 토글.

### 3.2. `StrategyBriefView`
* **설명**: 8단계 스토리보드 상세 명세를 출력하는 컴포넌트.
* **구성 요소**: 단계별 카드 그리드, 각 단계의 이미지 컨셉 프롬프트 텍스트, 이미지 시안 슬롯.

### 3.3. `AuditTimeline`
* **설명**: 감사 센터용 타임라인 목록 뷰 컴포넌트.
* **구성 요소**: 필터 헤더, 타임라인 노드 리스트, 변경 세부사항 모달 팝업 바인딩.

### 3.4. `CircuitStatusGauge`
* **설명**: 탄력성 모니터용 Redis Circuit Breaker 모니터링 컴포넌트.
* **상태**:
  - `CLOSED`: 정상 운용 (Green).
  - `OPEN`: 장애 격리 / 우회 Fallback 중 (Red).
  - `HALF_OPEN`: 부분 회복 자가진단 중 (Yellow).
