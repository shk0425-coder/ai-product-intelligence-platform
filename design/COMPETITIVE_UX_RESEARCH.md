# 경쟁 서비스 UX 리서치 (Competitive UX Research)

본 문서는 네이버 스마트스토어, 쿠팡 Wing, Canva, Notion, Figma 등 5개 서비스의 UX를 분석하여 본 플랫폼에 벤치마킹할 요소와 배제해야 할 요소를 정의한다.

---

## 1. 서비스별 UX 비교 매트릭스

| 대상 서비스 | 주 사용처 (Domain) | 장점 (Pros) | 단점 (Cons) | 본 플랫폼 벤치마킹 포인트 | 배제해야 할 UX 패턴 (Anti-Pattern) |
|:---|:---|:---|:---|:---|:---|
| **네이버 스마트스토어 센터** | 커머스 판매 채널 관리 | 친숙한 정형 데이터 목록 구성, 다양한 카테고리 설정. | 메뉴가 지나치게 많아 복잡하고, 모바일 뷰 최적화 부족. | 판매자가 가장 익숙한 목록 Grid 스타일 차용. | 3단계 이상의 깊은 아코디언 메뉴 구성 지양. |
| **쿠팡 Wing** | 커머스 판매 채널 관리 | 매출 성과 및 주요 알림을 최상단에 카드 형태로 시각화. | 일방적인 패널티 정책 중심 UI, 시각적 노이즈가 강함. | 대시보드 최상단의 '추천 조치 사항' 카드 구성법. | 사용자에게 불안감을 주는 붉은색 경고의 남용 금지. |
| **Canva** | 템플릿 기반 디자인 제작 | 고도로 간소화된 사이드바, '추천 템플릿' 중심의 첫 화면. | 자유도가 너무 높아 초보자가 빈 캔버스에서 시작할 때 방황. | "첫 추천 전략 기획안"을 템플릿 카드 형태로 바로 제안. | 빈 화면에서 사용자에게 모든 선택권을 일임하는 UI 배제. |
| **Notion** | 문서 및 데이터베이스 관리 | 드래그 앤 드롭을 통한 블록 조작, 슬래시(/) 명령어 편의성. | 데이터베이스 기능이 깊어질수록 학습 장벽이 기하급수적으로 상승. | 분석 결과를 접었다 펼 수 있는 토글(Toggle) 가독성 구현. | 상세 분석 테이블의 무한 스크롤 및 지나친 기능 중첩 배제. |
| **Figma** | 협업 디자인 도구 | 직관적인 캔버스 줌인/아웃, 레이아웃 그리드 시스템. | 일반 사무용 사용자 기준 너무 많고 무거운 단축키 및 툴바. | 상품 이미지와 썸네일을 시각적으로 정렬하는 카드 그리드 레이아웃. | 기능이 빼곡히 들어찬 복잡한 우측 속성 패널(Properties Panel). |

## 2. 핵심 영역별 UX 벤치마킹 분석

### 2.1. 탐색 및 네비게이션 (Navigation)
* **Canva & Notion**: 메뉴 개수를 6개 이내로 간소화하고, 하위 탐색은 컨텍스트 메뉴나 토글 탭으로 숨겨 인지 부하를 줄임.
* **적용**: 본 플랫폼의 글로벌 메뉴를 **[홈 / 상품 / 분석 / AI / 프로젝트 / 설정]**의 딱 6개로 통제하고, 모든 관리자 메뉴는 프로필 클릭 하위의 숨겨진 제어판 영역으로 격리함.

### 2.2. 대시보드 온보딩 (Onboarding)
* **Canva**: 첫 진입 시 빈 화면 대신 "무엇을 만들고 싶으신가요?"라는 검색창과 몇 가지 추천 카드를 노출하여 행동을 유도함.
* **적용**: 최초 로그인 시 빈 대시보드 대신 **"첫 번째 상품의 시장 분석을 시작하세요"**라는 눈에 띄는 온보딩 퀵스타트 카드를 강제 노출하고, 첫 완료 시 자동으로 가이드가 사라지게 설계함.

### 2.3. AI 상호작용 (AI Interaction)
* **Notion AI**: 단순 채팅창 방식 대신, 문서 컨텍스트 내에서 버튼 클릭을 통해 필요한 요약을 바로 얻을 수 있게 지원.
* **적용**: 판매자에게 챗봇 형태의 대화형 프롬프트 입력을 강제하지 않고, **"AI 상품 기획 요청"**, **"AI 썸네일 기획"**과 같이 명확히 정의된 단일 목적의 Action Button을 카드로 감싸 제공함.

## 3. 우리 제품이 절대 하지 말아야 할 UX (Anti-UX Goal)

1. **"분석 중..." 화면에서 무한 대기 유발**: 쿠팡 Wing이나 타 분석기처럼 진행률 표시 없이 로딩 스피너만 도는 UX는 사용자의 이탈을 가속화함. 예상 소요시간과 멀티스텝 진행 현황을 실시간 표시함.
2. **복잡한 통계 차트 남용**: 꺾은선, 막대, 원형 그래프가 화면을 덮는 네이버 스마트스토어 통계 뷰는 정보 과부하를 초래함. 그래프는 보조 수단으로 격하시키고, **"A등급 (소비자 만족도 85점으로 우수)"**과 같은 텍스트 기반 결론을 최상단에 배치함.
3. **확인 모달의 이중 사용**: 중요한 실행(예: 기획안 확정) 시 연속으로 확인 창을 2번 띄우는 복잡성을 완전히 배제하고, 원클릭 후 토스트 메시지로 보완함.
