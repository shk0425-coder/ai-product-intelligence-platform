# UX_PATTERN.md

# 공통 UX 디자인 패턴 (UX Patterns)

본 문서는 플랫폼 내에서 반복적으로 사용되는 주요 흐름과 상호작용에 대한 **표준 UI/UX 패턴 가이드**를 정의한다.

---

## 1. 비동기 작업 및 에러 복구 패턴 (Error Recovery & Retry)

네트워크 단절, 크롤링 차단, API 한도 초과 등 비동기 통신 실패 시 사용자에게 원인과 즉각적인 탈출 경로(Action)를 제공하는 것을 목표로 한다.

```text
+--------------------------------------------------+
|                    [에러 아이콘]                  |
|                                                  |
|          네이버 쇼핑 연결 상태가 불안정합니다.       |
|       네이버 쇼핑 일시적 부하 또는 네트워크 순단으로 |
|         분석이 중단되었습니다. (Error: TIMEOUT)   |
|                                                  |
|            +-------------------------+           |
|            |    다시 시도 (Retry)     |           |
|            +-------------------------+           |
+--------------------------------------------------+
```

### 1.1. 구성 규칙
* **상세 코드 노출 금지**: 날것의 Stack Trace나 개발자용 상세 에러는 클릭 시 펼칠 수 있는 디버그 영역(`디버그 로그 보기` 아코디언)으로 숨김 처리한다.
* **복구 CTA (Call to Action)**: 반드시 사용자가 즉시 조치를 취할 수 있는 주 행동 버튼(예: `다시 시도`, `상태 확인`)을 1개 이상 배치한다.
* **배경 복구(Retry Limit)**: UI 수준에서 자동으로 API 호출을 최대 3회 재시도(Exponential Backoff)해 보고, 그래도 실패하면 비로소 이 화면을 노출한다 (React Query의 `retry` 옵션 연계).

## 2. 리뷰 수집 & 분석 단계적 위저드 패턴 (Multi-step Wizard)

신규 상품 소싱 분석 시, 약 1분이 소요되는 작업을 투명하게 인지시키는 단계적 안내 패턴이다.

```text
[1단계: 리뷰 수집] ── (성공) ──> [2단계: 결핍 분류] ── (성공) ──> [3단계: 기획 등급 계산]
     (수집 중...)                     (대기 중)                        (대기 중)
```

### 2.1. 단계 구성
1. **수집 단계 (Scraping)**:
   - Playwright 기반 크롤러가 쇼핑 리뷰 데이터를 가져오는 단계.
   - UI: 로딩 바와 수집 속도 및 개수 표시 (`120/300개 리뷰 수집 중`).
2. **분석 단계 (Analyzing)**:
   - 수집된 원시 데이터 기반 LLM 감성/결핍 분석 단계.
   - UI: "AI가 소비자의 목소리에서 아쉬워하는 점을 찾고 있습니다" 안내 애니메이션.
3. **등급 및 저장 단계 (Grading & Save)**:
   - 백엔드 룰 엔진이 가중합을 계산해 최종 데이터베이스에 적재하는 단계.
   - UI: "마지막으로 상품 기획 가치 등급을 판별하고 있습니다."

## 3. 감사 로그 데이터 조회 패턴 (Pagination vs. Infinite Scroll)

감사 센터(Audit Center)는 매니저나 대표가 데이터 보존 기간 동안의 다량의 보안 기록을 검색하는 화면이다.

* **감사 로그 조회**: **페이지네이션(Pagination)을 표준으로 삼는다.**
  - **이유**: 감사 로그는 특정 일자, 특정 시간의 사건을 정확하게 추적하는 것이 목적이므로, 무한 스크롤(Infinite Scroll) 적용 시 스크롤 위치 제어가 어렵고 이전 위치로 돌아갈 수 없어 정보 탐색 효율이 크게 떨어진다.
  - UI 구성: 한 페이지당 20개 내외의 행을 노출하고, 하단에 `[처음] [<] [1] [2] [3] [>] [끝]` 형태의 표준 페이지네이터 배치.
* **상품 목록/기획안 목록**: **그리드 스켈레톤 및 페이지네이션 결합.**
